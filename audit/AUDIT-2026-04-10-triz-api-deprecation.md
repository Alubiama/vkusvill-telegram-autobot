# AUDIT-2026-04-10: VkusVill API Deprecation & TRIZ Ideation

## Суть инцидента
В ночь на 10 апреля 2026 года процессы сбора данных (задачи `collect-00:15` и `collect-00:30`) завершились с критической ошибкой: `ValueError: All collect sources failed`. 
Предыдущие изыскания агентов в других ветках проекта выявили, что сервер ВкусВилла на запросы старых эндпоинтов (таких как `user/privAbonement/update` и `abonementItems`) возвращает 404 и текст вида «Обновитесь до последней версии». Ошибочно считалось, что версия API бота `26.5.11` устарела.
Однако пробив по независимым каналам показал, что **26.5.11 — это и есть последняя версия Android-приложения** на апрель 2026 года. Таким образом, заглушка об обновлении означает полное закрытие/переименование данных эндпоинтов на стороне ВкусВилла. 

Данные о скидках теперь интегрированы непосредственно в Server-Driven UI ответ нового эндпоинта `abonementScreen`.

---

## Решения на основе ТРИЗ (Прогон 1: Оптимизация работы с текущим API)

**ИКР (Идеальный конечный результат):** Сервер сам отдает нам чистые данные в рамках грязного UI-ответа (abonementScreen), избавляя от написания нестабильных парсеров верстки.

1. **Поиск скрытого ресурса (ВПР):** Извлечение массива `resources` / `entities` прямо из корня JSON ответа `abonementScreen` (где чистые модели обычно хранятся отдельно от дерева верстки).
2. **Экстрактор (Вынесение):** Написание "слепого" рекурсивного парсера, который вытягивает любые объекты, имеющие ключи `productId` и `price`, игнорируя иерархию Server-Driven UI.
3. **MCP-Посредник (Копирование):** Извлечение из ответа `abonementScreen` ТОЛЬКО идентификаторов товаров (`productId`/`inshop_id`). Далее эти ID прогоняются через существующий `vkusvill_mcp_client.py` (ручка `product_details`), который выдает эталонные, красивые JSON данные.
4. **Слепое прокликивание (Наоборот):** Бот просто выполняет обнаруженные в UI шорткаты `action://update.abonement...` вслепую, добавляя всё доступное, и читает результат из корзины.

## Решения на основе ТРИЗ (Прогон 2: Смена парадигмы и переход в надсистему)

Вместо вечной "гонки вооружений" с защитой мобильного API, можно изменить сам подход к сбору данных.

1. **Переход в надсистему (Официальный Telegram-бот):** 
   У ВВ есть официальный бот (например, `@vkusvillbot`). Мы запускаем Userbot (скрипт на базе Telethon/Pyrogram с правами пользователя). Раз в сутки бот пишет официальному боту команду показа скидок и парсит полученное текстовое сообщение. Парсинг официального телеграм-ответа на порядки долговечнее и надежнее взлома APK.
2. **Push вместо Pull (Самообслуживание):** 
   Настроить email-информирование о скидках в профиле ВВ. Бот читает почту (IMAP) и парсит входящие письма. Почтовые шаблоны живут годами и не защищены от ботов.
3. **Итоговая корзина (Разделение в пространстве / Универсализация):** 
   Бот вообще перестает сканировать список скидок. Пользователь через Mini App набирает любые товары. Бот вбрасывает их в корзину ВкусВилла. Сама платформа ВкусВилла применяет все "Любимые продукты" и акции на этапе "Итого", а бот просто читает результирующую чековую строку (Корзина надежна). 
4. **Скрипт-самовосстановление (Динамичность):** 
   При падении с ошибкой 404, система автоматически выкачивает свежий APK с APKMirror, прогоняет через dexdump/strings, находит новые URL регулярными выражениями и делает hot-swap эндпоинтов в `providers.py`.
